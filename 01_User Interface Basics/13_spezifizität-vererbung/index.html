CSS-Kaskade, Spezifizität & Vererbung

CSS-Kaskade

Die CSS-Kaskade ist die Reihenfolge, in der die CSS-Regeln definiert sind. Später definierte Regeln bei gleicher Spezifizität überschreiben vorher definierte Regeln.

CSS-Spezifizät

Je spezifischer ein CSS-Selektor, desto wichtiger sind die in ihm definierten Regeln.

Regeln, die spezifischer definiert sind, überschreiben Regeln, die weniger spezifisch definiert sind.

Die exakte Spezifizität findet ihr hier heraus: Spezifizitäts-Rechner

Alle Regeln nach Spezifizität sortiert:

  1. !important:

    Ein mit `!important` markierter Style überschreibt alle anderen Stile immer

    Bitte NIEMALS benutzen

    Es gibt immer eine andere Lösung & es macht das CSS unübersichtlich.

  2. Inline CSS: Inline CSS überschreibt alle anderen Selektoren. Bitte nicht in der Praxis benutzen (höchstens zum Debugging).
  3. #id: Eine ID ist spezifischer als jegliche Anzahl an Klassen oder Elementen.
  4. .class1.class2: Zwei Klassen zu definieren als spezifischer als eine Klasse.
  5. .class: Eine Klasse zu definieren ist spezifischer als Elemente zu definieren.
  6. html body h1: Drei Elemente. Noch spezifischer als zwei.
  7. body h1: Zwei Elemente. Spezifischer als nur `h1`
  8. h1: Ein Element. Am wenigsten spezifisch

CSS-Vererbung

Einige CSS-Eigenschaften werden automatisch vererbt und andere nicht.

Übersicht über alle vererbten Eigenschaften

Vererbung testen

Vererbte Eigenschaften werden direkt an alle Kind-Elemente weiter gegeben. Nicht vererbte Eigenschaften werden nur fürs Element selbst gesetzt.

Vererbungsverhalten überschreiben

Wir können dieses Vererbungsverhalten für unserer Elemente auch überschreiben:

  • inherit: Eigenschaft vom Eltern-Element erben
  • unset: Vererbungsverhalten zurücksetzen (vererbte Eigenschaften werden wieder vererbt, nicht vererbte Eigenschafter werden nicht mehr vererbt.
  • initial: Styling auf den ursprünglichen Wert zurücksetzen (bevor der Browser überhaupt gestylt hat).
